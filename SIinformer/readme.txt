-------------------- Назначение программы --------------------
Программа отслеживает обновления авторов на самиздате.
Используются следующие признаки: новый текст, изменение размера, изменение описания.
1) Запихиваем в программу любимых авторов (в интерфейсе или импортом из SI tst) 
2) Засовывем исполнительный файл в автозагрузку (если хочется). 
3) При запуске прога проверяет обновления и потом делает это каждый час (можно и ручками запускать, есть там кнопка такая). 
4) Программа сидит в трее, если есть что-то новое у отслеживаемых авторов - в трее выскакивает сообщение, кто из них обновился. 
5) В основном окне программы в списке у авторов с обновлениями горит (не во лбу) желтая звезда (нет обновления - серая звездочка). 
6) При двойном щелчке на авторе открывается его информация или страничка (как настроите). По умолчанию - информация.
7) Все.
   
-------------------- Системные требования	 --------------------
Windows XP SP3, Windows Vista (возможно нужно SP1?).
.NET 3.5 SP1 (SP1 - обязательно).

-------------------- Основное --------------------
Программа отображает список авторов в 2-х режимах (переключаются клавокомбинацией Ctrl+E):
1. Простой - отображается имя автора, дата обновления и признак новых прозведений (желтая звезда)
2. Расширенный - отображаются дополнительные кнопки и возможности (см. ниже).
Такое разделение выполнено, чтоб не загромождать интерфейс в обычной жизни и иметь возможность поуправлять всеми возможностями программы при необходимости.
Желтая звезда является кликабельной и помечает автора и его произведения как прочитанные.

-------------------- Панель управления программы (над списком авторов) --------------------
Справа налево.
1. (стрелки) Смена направления сортировки на противоположное.
2. (а/д) Режим сортировки по автору(а)/по дате обновления(д).
3. (е) Режим программы простой/расширенный.
4. (с) Показывать/не показывать категории.
5. Фильтр. Отображаются только те авторы, которые в своем имени содержат введеную строку.
   Если фильтр начать с '~', то ищутся авторы, которые начинаются с указанного текста (например, для фильтра '~к' найдутся авторы с буквы к).

-------------------- Расширенный режим --------------------
В этом режиме есть возможность добавления автора (внизу окна).
Дополнительные кнопки для автора расширенного режима. Слева направо.
1. (i) Окно с информацией о произведениях.
2. (w) Страница автора на самиздате.
3. (с) Копировать информацию об авторе.
4. (r) Редактировать имя автора.
5. (cat) Сменить категорию для автора.
6. (u) Проверить обновления автора.
7. (x) Удалить автора из списка. ОСТОРОЖНЕЕ. Удаляет сразу, ничего не спрашивая.
8. Галка "не проверять". Автор не будет проверяться на обновления.
9. Галка "кешировать" (см. ниже в разделе "читалка").
В принципе все эти кнопки дублируются клавиатурой, за исключением категорий и копирование информации об авторе работает немного по другому (см. ниже "Клавиши управления").

Дополнительные кнопки для категории в расширенном режиме.
1. (r) Редактировать имя категории.
2. (x) Удалить категорию из списка, если она пустая.

-------------------- Категории --------------------
Автор может принадлежать одной категории. 
Категория выбирается кнопкой (cat) в расширенном режиме в панели автора. В открывшемся меню можно выбрать существующую категорию, или создать новую. При создании новой категории выбранный автор сразу туда переместится.
Добавленные авторы попадают в категорию по умолчанию "Default".
Категории можно сворачивать/разворачивать (кнопка слева плюс шорткаты), упорядочивать вручную (кнопки справа).
Категорию можно переименовать (кнопка (r) в расширенном режиме или шорткат).
Категорию можно удалить, если она пустая (кнопка (х) в расширенном режиме или шорткат).

-------------------- Читалка и конвертация текста --------------------
Возможно настроить 4 типа читалки. Встроенная и другая читалка кешируют скачанные файлы.
Максимальный размер кеша задается в настройках (0-без ограничений, 50 Мб-по умолчанию). 
Кеш чиститься при необходимости при запуске программы. Стратегия очистки кеша - удаление самых старых файлов.

Для настройки любой читалки задаются следующие параметры.
1) Читалка - путь к программе-читалке.
2) Параметры запуска - командная строка читалки, если она может понимать эту командную строку. 
   В командной строке может быть указано {0} - место, куда подставляется имя файла (скачанного или сконвертированного, если настроена конвертация). 
   Если строка пустая, подразумевается, что там только {0}.
3) Конвертер - программа, преобразующая скачанную html страницу в формат, понятный читалке (чистый txt, fb2 и другие).
4) Параметры запуска - командная строка конвертера. При пустой строке подразумевается {0} {1}.
   Кроме ключей конвертации, НЕОБХОДИМО ОБЯЗАТЕЛЬНО указать:
	  {0} - место, куда будет подставляться имя скачанного файла
	  {1} - место, куда будет подставляться имя сконвертированного файла.
	  или
	  {1|расширение} - место, куда будет подставляться имя сконвертированного файла с указанным расширением. Расширение указывается без точки {1|txt}.

Книги для читалки могут быть закешированы при проверки обновления.
Настройки кеширования устанавливаются на 3-х уровнях: для книги, для автора и для всей программы.
Для книги можно установить: кешировать, не кешировать, использовать настройки автора (серая галка).
Для автора можно установить: кешировать, не кешировать, использовать настройки программы (серая галка).
Для программы можно установить: кешировать, не кешировать.

-------------------- Клавиши управления --------------------
F1 - открыть информацию о произведениях.
F2 - открыть страничку автора на самиздате.
Ctrl+Delete - удалить автора или категорию из списка. ОСТОРОЖНЕЕ. Автор удаляется сразу, ничего не спрашивая. Категория должна быть пустая.
Ctrl+E - простой режим/расширенный режим (переключатель).
Ctrl+W - показывать/не показывать категории (переключатель).
Ctrl+С - копировать информацию об авторе (в простом режиме копирует имя и адрес, в расширенном информацию о новых произведения). Нужно для вставки в форум на кубиках :).
Ctrl+М - пометить автора и и его произведения как прочитанные.
Ctrl+I - игнорировать автора (переключатель).
Ctrl+R - переименовать автора или категорию.
Ctrl+F - фильтр.
Ctrl+U - запустить обновление текущего автора.
Ctrl+Shift+U - запустить/остановить обновление всех неигнорируемых авторов.
Ctrl+A - сортировка по автору
Ctrl+D - сортировка по дате
Ctrl+S - сортировка по автору/дате (переключатель)
Ctrl+стрелка вверх - сортировка по возрастанию
Ctrl+стрелка вниз  - сортировка по убыванию
Ctrl+Shif+стрелка вверх или Ctrl+Shift+стрелка вниз - сортировка по возрастанию/убыванию (переключатель)
стрелка влево на авторе - перейти на категорию
стрелка влево на категории - свернуть
стрелка вправо на категории - развернуть
двойной клик и Enter на категории - свернуть/развернуть
Ctrl+S - сохранить книгу
Ctrl+R - почитать книгу в выбранной в настройках читалке
F1,F2,F3,F4 - почитать книгу в встроенной читалке, в инете, в AJreader'е, в другой читалке соотвественно

-------------------- Дополнительные возможности (неочевидные и неневероятные) --------------------
1. Программа по результатам обновления генерирует rss-ленту. Сохраняется в папке программы в файле siinformer.rss.
   Допонительно понадобится:
   a) локальный web сервер (я пользую HFS 2.3 beta) 
   б) служба преобразования динамического IP в статический URL (dyndns, no-ip и др., возможно предоставляет провайдер)
   в) rss-клиент для чтения ленты.
   В результате вы сможете получать свои "новости самиздата" везде.
2. Перетаскивание ссылки автора на список авторов добавит автора в список.
3. Размеры окон с текстами (обычного и динамического) запоминаются для каждого автора отдельно.
   Кроме этого существуют следующие дополнительные возможности.
   Изменение размера с нажатым Ctrl меняет размер по умолчанию.
   Изменение размера с нажатым Ctrl и Shift сбрасывает размер всех окон.
4. Аналогичные правила для разделителя комментария автора и для положения обычного окна с текстами.
5. Автозакрытие (при запуске с ключом командной строки autoclose после проверки обновлений программа завершится).
6. Бекап файла списка авторов. Выполняется при запуске программы после удачной загрузки списка.

-------------------- FAQ --------------------
1. А может ли программа работать через прокси ?
   В настройках.
2. Дата обновления автора на сайте отличается от даты обновления в программе.
   Многие авторы в последнее время используют самиздат в режиме блокнота и не обновляют дату на сайте. В случае всяческих косяков ставится текущая дата компьютера.
   Если есть корректная дата на странице автора, то берется она.
3. Автор пишет, что обновил текст zz.xx.yyyy, а в программе z1.x1.yyy1.
   Дата обновления текста получается не с сайта, а по локальному времени компьютера в момент обнаружения обновления.
4. Меня это окно с текстами так достает, как его убрать?
   В настройках.
